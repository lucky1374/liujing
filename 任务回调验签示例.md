# 任务回调验签示例

适用范围：AI 自动化测试平台任务回调（`event=task.execution.completed`）。

## 1. 回调头字段

当后端配置了 `TASK_CALLBACK_SECRET` 时，回调请求会带：

- `X-Task-Signature`: `sha256=<hex>`
- `X-Task-Timestamp`: 毫秒时间戳（字符串）
- `X-Task-Signature-Alg`: `HMAC-SHA256`

签名基串：

```text
<timestamp>.<raw_body>
```

签名算法：

```text
HMAC-SHA256(secret, signature_base)
```

## 2. 验签流程建议

1. 读取原始请求体（raw body，不要先 JSON 序列化再参与签名）。
2. 从 header 取 `X-Task-Timestamp`、`X-Task-Signature`。
3. 用本地同一个 `TASK_CALLBACK_SECRET` 重新计算签名。
4. 使用常量时间比较函数比对签名。
5. 校验时间戳窗口（建议 5 分钟内），防重放。

## 3. Node.js 示例（Express）

```js
import crypto from 'crypto'
import express from 'express'

const app = express()
const secret = process.env.TASK_CALLBACK_SECRET || ''

app.use(express.json({
  verify: (req, _res, buf) => {
    req.rawBody = buf.toString('utf8')
  }
}))

function verifyTaskCallback(req) {
  const signatureHeader = req.header('X-Task-Signature') || ''
  const timestamp = req.header('X-Task-Timestamp') || ''
  const rawBody = req.rawBody || ''

  if (!secret) {
    throw new Error('TASK_CALLBACK_SECRET is empty')
  }
  if (!signatureHeader.startsWith('sha256=')) {
    throw new Error('invalid signature header format')
  }

  const received = signatureHeader.slice('sha256='.length)
  const base = `${timestamp}.${rawBody}`
  const expected = crypto.createHmac('sha256', secret).update(base).digest('hex')

  const valid = crypto.timingSafeEqual(Buffer.from(received, 'hex'), Buffer.from(expected, 'hex'))
  if (!valid) throw new Error('signature mismatch')

  const now = Date.now()
  const ts = Number(timestamp)
  const maxSkew = 5 * 60 * 1000
  if (!Number.isFinite(ts) || Math.abs(now - ts) > maxSkew) {
    throw new Error('timestamp expired')
  }
}

app.post('/webhook/task-callback', (req, res) => {
  try {
    verifyTaskCallback(req)
    // 通过验签后再处理业务
    res.status(200).json({ ok: true })
  } catch (error) {
    res.status(401).json({ ok: false, message: error.message })
  }
})

app.listen(9010)
```

## 4. Python 示例（FastAPI）

```python
import hashlib
import hmac
import os
import time

from fastapi import FastAPI, Header, HTTPException, Request

app = FastAPI()
SECRET = os.getenv("TASK_CALLBACK_SECRET", "")


def verify_signature(raw_body: bytes, timestamp: str, signature_header: str):
    if not SECRET:
        raise HTTPException(status_code=500, detail="TASK_CALLBACK_SECRET is empty")
    if not signature_header.startswith("sha256="):
        raise HTTPException(status_code=401, detail="invalid signature header format")

    received = signature_header.replace("sha256=", "", 1)
    base = f"{timestamp}.{raw_body.decode('utf-8')}".encode("utf-8")
    expected = hmac.new(SECRET.encode("utf-8"), base, hashlib.sha256).hexdigest()

    if not hmac.compare_digest(received, expected):
        raise HTTPException(status_code=401, detail="signature mismatch")

    now_ms = int(time.time() * 1000)
    ts_ms = int(timestamp)
    max_skew = 5 * 60 * 1000
    if abs(now_ms - ts_ms) > max_skew:
        raise HTTPException(status_code=401, detail="timestamp expired")


@app.post("/webhook/task-callback")
async def task_callback(
    request: Request,
    x_task_signature: str = Header(default=""),
    x_task_timestamp: str = Header(default=""),
):
    raw_body = await request.body()
    verify_signature(raw_body, x_task_timestamp, x_task_signature)

    payload = await request.json()
    # 验签通过后处理 payload
    return {"ok": True, "event": payload.get("event")}
```

## 5. 排障建议

- 签名不通过优先检查：
  - 接收方是否使用了 raw body（而不是重新 JSON stringify）
  - 时间戳窗口是否过窄
  - 两边 `TASK_CALLBACK_SECRET` 是否一致
- 如果回调记录持续失败：
  - 在平台任务页查看“回调记录”与“回调健康告警”
  - 使用“重试回调/重试失败回调”验证恢复情况
