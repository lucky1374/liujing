# 本地启动说明

## 1. 适用场景

本文档适用于无法安装 Docker Desktop 的开发环境，使用本机直接运行 AI 自动化测试平台。

## 2. 启动前准备

需要确保本机已具备：

- Node.js
- npm
- Python3
- MySQL
- Redis

并且：

- 后端 `.env` 已正确配置
- MySQL 数据库已创建
- Redis 已启动

## 3. 一键启动脚本

仓库根目录已提供：

- `start-local.sh`
- `stop-local.sh`

## 4. 启动方式

```bash
cd <你的项目根目录>
chmod +x start-local.sh stop-local.sh
./start-local.sh
```

启动后：

- 后端 Swagger：`http://localhost:3000/api-docs`
- 前端页面：`http://localhost:5173`

## 5. 停止方式

```bash
cd <你的项目根目录>
./stop-local.sh
```

## 6. 日志位置

日志目录：

- `<项目根目录>/.local-run/backend.log`
- `<项目根目录>/.local-run/frontend.log`

## 7. 首次启动注意事项

- 脚本会自动检查并初始化 `python-runner/.venv`
- 不会自动初始化 MySQL 或 Redis
- 如管理员账号尚未创建，需要单独执行：

```bash
cd /Users/lj/liujing/ai-test-platform
npm run seed
```

## 8. 建议使用方式

当前阶段推荐直接使用本地直跑方式作为：

- 日常开发方式
- 联调方式
- 内部演示方式

原因：

- 已经稳定跑通
- 不依赖 Docker Desktop
- 对当前 MVP 阶段最轻量、最稳定

## 9. 通用性说明

当前 `start-local.sh` 和 `stop-local.sh` 已改为基于脚本所在目录自动推导项目根目录。

这意味着：

- 不再依赖固定的 `/Users/lj/liujing`
- 只要目录结构保持不变，就可以在其他机器上直接复用
