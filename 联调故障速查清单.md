# 联调故障速查清单

适用范围：AI 自动化测试平台 v0.2（NestJS 后端 + python-runner + Web 前端）。

## 一、先做 30 秒健康检查

```bash
# 1) Runner 是否在线
curl -i http://localhost:8001/health

# 2) 8001 端口是否有进程监听
lsof -i :8001 -sTCP:LISTEN -n -P

# 3) 后端环境变量是否已配置（在后端启动终端执行）
printenv | grep -E "PYTHON_RUNNER_URL|RUNNER_AUTH_TOKEN"
```

预期：

- `/health` 返回 `HTTP 200`
- 8001 有监听进程
- 后端能看到 `PYTHON_RUNNER_URL`，并且 `RUNNER_AUTH_TOKEN`（如启用鉴权）已配置

## 二、平台内置诊断入口

- 页面：`测试任务` -> `Runner诊断`
- 接口：`GET /test-tasks/runner/diagnostics`
- 接口：
  - `GET /test-tasks/runner/diagnostics`
  - `GET /test-tasks/observability/overview?hours=24`
- 返回项：
  - `runnerUrl`
  - `hasAuthTokenConfigured`
  - `health`（/health 检查）
  - `executeProbe`（/execute 探测）
  - `statsProbe`（/stats 探测）

页面增强：

- 任务页已支持 Runner 实时状态卡（并发上限、执行中、排队中、排队拒绝）

- 观测概览支持时间范围切换：近 1 小时 / 近 24 小时 / 近 7 天

建议：联调群反馈问题时，优先粘贴“复制诊断信息”的结果。

## 三、高频故障 -> 处理动作

### 1) 执行来源仍是“Python 本地兜底”

常见原因：

- HTTP Runner 调用失败后触发兜底
- `PYTHON_RUNNER_URL` 错误或 Runner 未启动

排查：

```bash
curl -i http://localhost:8001/health
curl -H "Authorization: Bearer <YOUR_JWT_TOKEN>" http://localhost:3000/test-tasks/runner/diagnostics
```

### 2) Python HTTP Runner 鉴权失败（401/403）

常见原因：

- 后端与 Runner 的 `RUNNER_AUTH_TOKEN` 不一致

排查：

```bash
# 查看 Runner 进程环境（macOS）
ps eww -p $(lsof -ti :8001)

# 检查后端 .env
grep -E "PYTHON_RUNNER_URL|RUNNER_AUTH_TOKEN" /Users/lj/liujing/ai-test-platform/.env
```

修复：

- 将两边 token 改成完全一致
- 重启后端与 Runner

### 3) ECONNREFUSED / connect refused

常见原因：

- Runner 未启动
- 地址或端口错误

排查与修复：

```bash
cd /Users/lj/liujing/ai-test-platform/python-runner
source .venv/bin/activate
uvicorn runner.server:app --host 0.0.0.0 --port 8001
```

### 4) ENOTFOUND / EAI_AGAIN

常见原因：

- `PYTHON_RUNNER_URL` 域名写错
- 本机 DNS 异常

动作：

- 优先改为 `http://localhost:8001` 验证链路
- 再按环境需要切回域名地址

### 5) timeout / ETIMEDOUT

常见原因：

- Runner 负载高或脚本执行慢
- 网络不稳定

动作：

- 先复测 `/health` 与 `/execute` 探针
- 观察 Runner 日志
- 必要时提高 Runner 调用超时时间（后端当前为 35000ms）

### 6) queue_timeout / HTTP 429

常见原因：

- Runner 并发满载，排队等待超时

动作：

- 查看 `/stats` 中 `running/waiting/rejected`
- 按需提高 `RUNNER_MAX_CONCURRENCY`
- 按需提高 `RUNNER_QUEUE_TIMEOUT_MS`

### 7) python_local + No module named

常见原因：

- python-runner 虚拟环境缺依赖

修复：

```bash
cd /Users/lj/liujing/ai-test-platform/python-runner
source .venv/bin/activate
pip install -r requirements.txt
python -m runner.main --payload examples/sample_payload.json
```

## 四、标准启动命令（本地联调）

### 1) 启动 Runner

```bash
cd /Users/lj/liujing/ai-test-platform/python-runner
source .venv/bin/activate
export RUNNER_AUTH_TOKEN=<YOUR_RUNNER_AUTH_TOKEN>
export RUNNER_MAX_CONCURRENCY=2
export RUNNER_QUEUE_TIMEOUT_MS=10000
uvicorn runner.server:app --host 0.0.0.0 --port 8001
```

### 2) 启动后端

```bash
cd /Users/lj/liujing/ai-test-platform
npm run start:dev
```

### 3) 启动前端

```bash
cd /Users/lj/liujing/ai-test-platform-web
npm run dev
```
